
	グループ式
	
※グループ式という名称は、上大が勝手に付けたものです。

lex:
	- idx ++ が多すぎる
	
	(, ) を処理するように case 節を追加。
	
	なお、この時点で ctype に、')' が式区切りになることを書き加えておく。
	
enc:
	(, ) は値もタイプも変化しないので、他の記号と同じように case 節を追加。
	
prs:
	'(' は文頭となる。
	'(' から始まる値は、prs_expr_val で処理する。そのため、prs_expr_val は
		_nextType が int か '(' かで分類する switch の命令になる。
		'(' を読んだ後は、また prs_expr を行い、式を処理する。
		その後 ')' を読み飛ばして解析完了。
		
	構文木上、グループ式は、PrsNodeType_Expr のノードになる。
	
	* ここで、抽象構文木(AST = Abstract Syntax Tree)についても触れる？
	
	dbglog で expr を処理できるようにしておくことを忘れない。
	
	なお、ここでエラー PrsErr_Paren が追加される。
	
cmp:
	式木のトラバース中に PrsNodeType_Expr が表れることを考慮するように。
	処理は単に、その子をルートとする部分木を、式木として処理し続けるだけ。
